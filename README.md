## Introduction

### What is ao?

`ao` is an `actor oriented` machine that emerges from a network of nodes running on the Arweave network. It represents a unified computing environment, or a Single System Image, facilitated by a heterogenous set of nodes in a distributed network.

---

## Key Features

- **Parallel Process Handling**: Supports an arbitrary number of parallel processes, connected through an open message passing layer.
- **Unbounded Computation**: No protocol-enforced limitations on computation size, maintaining network verifiability and trust minimization.
- **Modular Architecture**: Allows integration with existing smart contract platforms, offering flexibility in virtual machines, message passing security, and payment options.
- **Global Unified Computing**: A global computing environment shared among all users, operating trustlessly and at any scale.

## Getting Started


## Contributing

Conventional commits are used for this repository.

To choose a task or make your own, do the following:

1. Add an issue for the task and assign it to yourself or comment on the issue.
2. Make a draft PR referencing the issue.

The general flow for making a contribution:

1. Fork the repo on GitHub.
2. Clone the project to your own machine.
3. Commit changes to your own branch.
4. Push your work back up to your fork.
5. Submit a Pull request so that we can review your changes.

**NOTE**: Be sure to merge the latest from "upstream" before making a pull request!

You can find tasks on the project board or create an issue and assign it to yourself.
